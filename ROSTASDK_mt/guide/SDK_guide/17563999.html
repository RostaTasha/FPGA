Руководство пользователя SDK RC-47 1.0 : AXI подсистема

Created by Yuri Rumyantsev, last modified by Sergey Torchigin on Apr 27, 2015

Внешняя DDR3 память, установленная на модуле EM4-DDR3 подключается к двум "банкам" ПЛИС. Для каждого "банка" внутри ПЛИС реализован свой контроллер на базе Xilinx IP ядра MIG - Top и Bottom. Эти контроллеры подключаются к внутрикристалльной шине AXI, на которой они имеют адреса

  • TOP: 0x20000000 - 0x3FFFFFFF (512 MB)
  • BTM: 0x40000000 - 0x5FFFFFFF (512 MB)

Для доступа к AXI шине через PCI Express используется ядро Stream-AXI Bridge, основанное на Xilinx Datamover IP и управляющей логики собственной разработки. Адресное пространство AXI не отображено в память хост-компьютера через BARы, вместо этого используется протокол записи/чтения через очереди RX_FIFO и TX_FIFO (использующиеся для передачи данных хост-ПЛИС в режиме DMA). Данный протокол реализуется совместно использованием ядра Stream-AXI Bridge и функций библиотеки API RC-47

fpga_write_to_axi()

fpga_read_from_axi()

 

AXI подсистема создается автоматически при создании Vivado проекта с помощью файла ./src/create_bd.tcl, который вызывается сценарием ./src/create_project.tcl проекта rc_pcie_ddr_2000_12.
