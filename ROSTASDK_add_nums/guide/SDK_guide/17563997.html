Руководство пользователя SDK RC-47 1.0 : Программное обеспечение ПЛИС Virtex-7

Created by Yuri Rumyantsev, last modified by Natasha Abrashitova on Jul 20, 2015

В данном разделе описываются проекты ПЛИС Virtex-7, входящие в SDK RC-47 версии 1.0.

Название проектаОписание проета
rc_pcie_base_2000_9 базовый ПЛИС проект, являющийся стартовым при включении питания блока. На этом проекте работают прикладные программы rc47_pcie_test и rc47_pcie_speed_test
rc_pcie_ddr_2000_12 ПЛИС проект, использующийся для тестирования памяти на модуле EM4-DDR3. Включает в себя внутрикристальную шину AXI, созданную с помощью инструмента Vivado Block Design. На этом проекте работает прикладная программа rc47_axi_ddr
rc_pcie_stress_2000_8_clk100_def_ff_u82 ПЛИС проект для работы прикладной программы rc47_stress_test

 

Все проекты создаются в среде Xilinx Vivado 2014.2. Сборка и отладка проектов производилась в ОС Windwos 7 64bits.  

Проект rc_pcie_stress_2000_8_clk100_def_ff_u82 распространяется в виде стандартного Vivado GUI проекта. В противоположность этому rc_pcie_base_2000_9 и rc_pcie_ddr_2000_12 поставляются в виде только исходных файлов, а также TCL скрипта для автоматического создания GUI проекта и имеют следующую структуру файлов в директории.

  • Директория src - место расположения исходных RTL, XDC и XCI (IP ядра) файлов
  • Директория project - в этой директории создается GUI проект Vivado
    • файл create_project.tcl - TCL сценарий автоматически создает Vivado проект и добавляет в него файлы из директории ../src
    • файл create_project.bat - сценарий windows, открывающий Vivado в batch режиме и вызывающий файл  create_project.tcl

 

Помимо этого проект rc_pcie_base_2000_9 может собираться по методологии Design Reuse (см. Vivado Design Suite User Guide Hierarchical Design UG905, Vivado Design Suite Tutorial Hierarchical Design UG946), когда из всего ПЛИС проекта выделяется интерфейсная статическая часть (Base IP) и пользовательская логика (User IP). Base IP не меняется при изменении User IP и его физического размещение на кристалле может быть фиксировано для 

  • гарантии, что в base ip не будет временных ошибок при последующих сборках проекта
  • ускорении сборки проекта

Для поддержки этого режима сборки в директории проекта есть следующие дополнительные директории и файлы

  • Директория Checkpoint - место расположения файла rosta_route_design.dcp, содержащего информацию о физического размещении блока Base IP. 
  • Директория tcl - набор TCL сценариев, использующихся для сборки проекта по методологии Design Reuse
  • файл design.tcl - исполняемый TCL сценарий "верхнего" уровня, управляющий сборкой проекта
  • файл run_design_tcl.bat - сценарий Windows, вызывающий Vivado в batch режиме и исполняющий design.tcl
  • файл readme_tcl.txt - короткая инструкция по работе с проектом по методологии Design Reuse

 

Далее в этом разделе описывается интерфейсная часть (Base IP), входящая во все ПЛИС проекты, а также подсистема AXI шины для поддержки внешней памяти на моделу EM4-DD3, входящая в проект rc_pcie_ddr_2000_12.
